# SP2 — UI React — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task.

**Goal:** Reemplazar la UI Jinja por una SPA React modular (Enhance / Models / Realtime-comingsoon / Settings) con cola de jobs en vivo, servida por FastAPI, con paridad funcional total + los módulos nuevos de SP1.

**Architecture:** `frontend/` Vite + React 18 + TypeScript + Tailwind + TanStack Query; build a `frontend/dist`, montado por FastAPI como SPA fallback (patrón bipolar-code). Dev: proxy Vite → :8090. Cliente API tipado consume las rutas REST existentes.

**Tech Stack:** Vite, React 18, TypeScript, TailwindCSS, @tanstack/react-query, react-router-dom, lucide-react (iconos), vitest + @testing-library/react.

**Spec:** docs/superpowers/specs/2026-07-14-upflow-v2-ux-design.md (§SP2 — binding).

## Global Constraints

### Design system (BINDING — "sala de control técnica", no template)
- **Dirección**: instrumento de precisión / control-room técnico. Denso pero calmado, data-forward, controles táctiles. NO glassmorphism, NO gradientes pesados, NO dark-SaaS genérico.
- **Paleta (tokens semánticos CSS vars + tailwind theme, NUNCA hex crudo en componentes)**:
  - Base grafito: `--bg` #0E0F13, `--surface` #16181F, `--surface-2` #1E212B, borde `--border` #2A2E3A (1px, la elevación viene de borde+tint, no blur).
  - Texto: `--text` #E6E8EE (≥4.5:1 sobre surface), `--text-dim` #9AA1B2 (≥3:1), `--text-faint` #6B7280.
  - Acento primario ámbar/señal: `--accent` #F5A524 (acción, progreso activo, CTA único por vista). Hover `#FFB84D`, press `#D98E12`.
  - Semánticos: éxito/done teal `--ok` #2DD4BF, error `--danger` #F04438, warn `--warn` #F5A524.
  - Modo claro NO es objetivo v2 (app desktop, dark por diseño deliberado — permitido por design-quality rules al ser decisión explícita). Un solo tema oscuro pulido.
- **Tipografía**: Space Grotesk (headings/UI, 500-700), Inter (body/labels, 400-500), JetBrains Mono **tabular** para TODO número (dims, fps, %, VRAM, tamaños, timings) — regla number-tabular crítica. Escala: 12/14/16/18/24/32. Line-height body 1.5. Cargar con font-display swap; self-host o Google Fonts (self-host preferido para el binario offline).
- **Efectos**: radio 8px (btn/card/input), 6px inputs pequeños; sin sombras salvo overlays/modales (scrim 50% black); micro-interacciones 150-250ms transform/opacity solo; progreso = movimiento protagonista (barras/anillos en job cards). prefers-reduced-motion respetado.
- **Layout**: DESKTOP-FIRST (app local en navegador a anchos desktop). Sidebar izquierda persistente (nav: Enhance / Models / Realtime / Settings, icono+label lucide, estado activo resaltado), área principal, panel derecho de cola de jobs con progreso en vivo. Responsive hasta ~900px (colapsa sidebar a iconos). Grid 8px. max-width contenido consistente.
- **A11y**: contraste AA verificado en ambos pares fg/bg; focus rings visibles 2px `--accent`; labels visibles en inputs; iconos-solo con aria-label; navegación teclado; color nunca único indicador (icono+texto en estados).
- **Iconos**: lucide-react SVG únicamente, NUNCA emoji. Stroke consistente 1.5-2px.

### Ingeniería
- TDD donde hay lógica (cliente API, hooks, transforms, componentes con estado). vitest. NO Playwright en esta iteración (smoke manual en T final).
- La UI Jinja (`app/templates/`, `app/web/`) vive hasta paridad; se elimina en la última tarea, reemplazando tests de template por tests de API + build check.
- El build de frontend NO corre en la suite pytest; se verifica aparte (`npm run build`). pytest sigue verde (535 tests) — las tareas que tocan backend (SPA serving, quitar Jinja) mantienen pytest verde.
- Commits español convencional; sin Co-Authored-By. Rama `feature/sp2-ui-react`.
- El binario/instalador: el build de `frontend/dist` debe incluirse en el paquete (SP3 lo empaqueta; package-release.ps1 gana `npm ci && npm run build` antes de zipear).
